# Mock Campaign Website Template

This project contains a simple, static website template designed for a class assignment or a mock political campaign. You can edit the HTML files directly to customize the content, names, and issues to match your candidate's platform.

## File Structure

| File | Description |
| --- | --- |
| `index.html` | The main landing page with a hero section, mission statement, and overview of key issues. |
| `about.html` | A biography page where you can describe the candidate's background and accomplishments. |
| `issues.html` | A page to outline your campaign’s policy positions on various issues. |
| `volunteer.html` | A volunteer sign‑up form encouraging supporters to get involved. |
| `contact.html` | A page with contact information so people can reach your campaign. |
| `styles.css` | A stylesheet that applies consistent styling across all pages. |

## Customization

* Replace the placeholders such as `[Candidate Name]`, `[Year]`, and the bracketed sections in each HTML file with your own text. 
* Adjust the navigation links if you add or remove pages. 
* Modify `styles.css` to change colors, fonts, and layouts. 
* Add images or logos by placing the image files in the same directory and referencing them in your HTML. 

## Viewing the Site

Open `index.html` in a web browser to view the home page. Use the navigation links to explore other pages. If you make changes to any files, refresh the browser to see your updates.

## Hosting

If you want to make your site accessible online, consider using free hosting options like [GitHub Pages](https://pages.github.com/), [Netlify](https://www.netlify.com/), or [GitLab Pages](https://docs.gitlab.com/ee/user/project/pages/). Each service has step‑by‑step guides for deploying static websites. Typically, you would:

1. Sign up for an account with the hosting provider. 
2. Create a new repository or project and upload the contents of this folder. 
3. Follow the provider’s instructions to enable static site hosting.

For example, on GitHub Pages you can create a repository named `your-username.github.io` and push these files to it. GitHub will automatically host the site at `https://your-username.github.io`. For Netlify, you can drag the folder onto their dashboard to deploy.

Remember to follow your instructor’s requirements for the assignment and feel free to adjust the design and structure as needed.
